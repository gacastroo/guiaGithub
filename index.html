Cómo registrarse en GitHub

  1. Navega a github.com en tu navegador
  2. Haz clic en Registrarse o en Continue with Google para usar tu cuenta de Google
  3. Sigue las indicaciones para crear tu cuenta personal
  4. Verifica tu dirección de correo electrónico Obligatorio

Cosas importantes

Algunas empresas crean cuentas de usuario administradas para sus empleados. No puedes registrarte con un correo que ya esté vinculado a una cuenta administrada.

Si tienes problemas para verificar tu correo, consulta la guía Verificar tu dirección de correo electrónico dentro de la documentación oficial de GitHub.

GIT VS GITHUB: La analogía más fácil

"Git es el cerebro 🧠, GitHub es la red social 🌐."

"Git guarda tu código, GitHub lo presume."



Git

  • Software local (en tu ordenador)
  • Control de versiones
  • Guarda instantáneas del proyecto
  • Ideal para uso individual
  • Funciona sin internet

GitHub

  • Plataforma web
  • Colaboración en equipo
  • Basado en Git
  • Gestión de proyectos
  • Red social para código

¿Qué es el control de versiones?

Un sistema que permite gestionar y rastrear los cambios en el código fuente a lo largo del tiempo, facilitando la colaboración y el desarrollo eficiente de software.

Estos son algunos de los comandos básicos de git y los que posiblemente mas utilices, espero que te ayuden :D
Haz clic en cualquier comando para copiarlo al portapapeles.

1 · Configuración inicial

git config --global user.name "Tu Nombre"
Configura tu nombre de usuario
git config --global user.email "tu@email.com"
Configura tu correo electrónico

2 · Iniciar un proyecto

git init
Crea un repositorio nuevo en la carpeta actual
git remote add <nombre> <url>
Crea una nueva conexión a un repositorio remoto.
git clone <url>
Descarga un repositorio existente

3 · El ciclo diario

git status
Muestra qué ha cambiado
git add .
Prepara todos los cambios para el commit
git commit -m "mensaje claro"
Guarda los cambios en el historial
git push origin main
Sube los cambios a GitHub
git pull origin main
Baja los cambios del remoto

4 · Ramas

git switch -c mi-rama
Crea y cambia a una rama nueva
git switch main
Vuelve a la rama principal
git merge mi-rama
Une la rama con la actual

5 · Si algo va mal

git log --oneline
Ver historial de commits resumido
git restore <archivo>
Descartar cambios en un archivo
git restore --staged <archivo>
Quitar un archivo del staging

Miniguía: ¿Dónde vive tu código?


Para trabajar con GitHub, imagina que tienes dos "copias" de tu proyecto:


  • Repositorio Local: Es la carpeta en tu ordenador. Es donde editas tus archivos día a día. GitHub no ve esto hasta que tú lo envías.
  • Repositorio Remoto: Es la copia en los servidores de GitHub. Es tu respaldo en la nube y el lugar donde los demás ven tu código.

¿Cómo se conectan?

Tú trabajas en tu Local. Cuando quieres que GitHub se entere, haces un commit (guardar foto instantánea) y luego un push (enviar a la nube).

☁️ Ahora está en internet

1. Crear el repositorio en GitHub

  1. Inicia sesión en GitHub.com.
  2. Haz clic en el icono "+" (esquina superior derecha) > "New repository".
  3. Ponle un nombre y elige si será Public o Private.
  4. Marca "Add a README file" (fundamental para documentar el proyecto).
  5. Haz clic en "Create repository".

2. Trabajar profesionalmente (VS Code + GitHub)

Vincular GitHub con VS Code:

  • Instala la extensión "GitHub Pull Requests and Issues".
  • Haz clic en el icono de usuario (abajo a la izquierda) > "Sign in to GitHub" y autoriza la conexión.

Hacer cambios (Flujo diario):

  1. Edita tus archivos y guárdalos.
  2. Ve a la pestaña de Source Control (icono de nodos a la izquierda).
  3. Haz clic en el "+" para preparar los cambios (Stage).
  4. Escribe un mensaje de commit y presiona el botón Commit.
  5. Haz clic en Sync Changes (o Push) para enviar todo a GitHub.

3. Alternativa: Línea de comandos (Terminal)

Para usar la terminal en VS Code, presiona Ctrl + Ñ o ve al menú Terminal > New Terminal. Una vez abierta, ejecuta estos comandos en orden:

git add . # Prepara todos los archivos para subir
git commit -m "Tu mensaje"  # Crea el punto de guardado (commit)
git push origin main # Sube los cambios al repositorio remoto

Nota: Asegúrate de estar dentro de la carpeta de tu proyecto en la terminal antes de escribir estos comandos.
